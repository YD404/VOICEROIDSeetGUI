# 開発トラブルシューティング・遅延原因レポート

本プロジェクトのフロントエンド構築において、想定よりも時間がかかった主な要因と、直面した技術的課題について報告します。

## 1. 概要
React + Vite + Tailwind CSS の環境構築自体はスムーズに進みましたが、実装後の動作確認フェーズで「画面が白紙になる」「日本語が文字化けする」という2つの大きな問題が発生し、その切り分けと修正に時間を要しました。

## 2. 直面した主な課題と原因

### ① 文字エンコーディングの不整合 (Shift-JIS vs UTF-8)
*   **現象**: コンポーネント内の日本語（「話者」「背景」など）がブラウザで `?` に化け、構文エラーやランタイムエラーを引き起こしました。
*   **原因**: Windows環境において、一部のファイル書き込みツールやPowerShellのデフォルト動作により、ファイルがBOMなしShift-JIS等で保存されていました。Vite（esbuild）がこれらを正しくUTF-8として解釈できず、内部コードが破壊されました。
*   **対策**: 
    *   PowerShellを使用して全ファイルをUTF-8（BOMなし）で強制的に再変換。
    *   さらなる安定性の確保のため、ソースコード内の日本語文字列を Unicode エスケープシーケンス（例: `\u8A71\u8005`）に置き換えて書き込み直しました。

### ② JSX ランタイムの衝突 (Hono vs React)
*   **現象**: ブラウザコンソールに `Objects are not valid as a React child` というエラーが表示され、レンダリングが失敗しました。
*   **原因**: 本プロジェクトのルートにある `tsconfig.json` が、バックエンド（Cloudflare Pages Functions / Hono）向けに `"jsxImportSource": "hono/jsx"` と設定されていました。Viteの開発用トランスフォーマ（esbuild）がこの設定を拾ってしまい、ReactコンポーネントをHonoのJSXとして処理しようとしたため、ランタイムエラーが発生しました。
*   **対策**: `vite.config.ts` の `@vitejs/plugin-react` 設定において、明示的に `{ jsxImportSource: "react" }` を指定し、ルートの `tsconfig.json` 設定を上書きすることで解決しました。

## 3. 時系列・作業ログの要約

1.  **環境構築**: パッケージインストール、Vite/Tailwind設定（順調）。
2.  **実装**: 全5コンポーネントのコード作成（順調）。
3.  **トラブル発生**: 起動直後、画面が白紙。APIのプロキシ問題か、コードの問題かの切り分け。
4.  **エンコード修正**: 文字化けを発見し、Unicodeエスケープによる堅牢な再実装。
5.  **JSX修正**: コンソールエラーからHonoとの衝突を特定し、Vite設定を修正。
6.  **最終確認**: 表示およびAPI連携の正常動作を確認。

## 4. 今後同様の事象が発生した際のチェックリスト（教訓）

今後、フロントエンドの構築や修正で問題（特に画面が表示されない、動作が不安定など）が発生した際は、以下の順序で「根本原因」を切り分けることを推奨します。

### ① ブラウザコンソールのエラーを最優先で確認する
*   **オブジェクトエラー (`Objects are not valid as a React child`)**: 今回のように、ランタイム（JSX等）の衝突が疑われます。使用しているフレームワークやライブラリの設定が干渉していないか確認してください。
*   **構文エラー (`SyntaxError`)**: 文字エンコーディングや、トランスパイルの失敗が疑われます。

### ② 最小構成での動作テスト（原因の切り分け）
*   複雑なコンポーネントを配置する前に、`App.tsx` や `main.tsx` を `<div>Hello</div>` だけにするなど、最小限のコードで描画されるか試してください。
    *   **描画される場合**: 実装したロジックやデータフェッチのコードに問題があります。
    *   **描画されない場合**: `tsconfig.json`、`vite.config.ts`、依存関係、またはトランスパイル設定など、プロジェクトの「根本」に問題があります。

### ③ 共有設定ファイルの干渉を確認する
*   同一リポジトリ内にバックエンド（Hono/Worker等）とフロントエンド（React/Vite等）が同居している場合、ルートの `tsconfig.json` や `package.json` が意図せずフロントエンドの設定に影響を与えていないか確認してください。
*   特に `jsxImportSource` や `types`、`moduleResolution` などの項目は要注意です。

### ④ ファイルのエンコーディングを確認する
*   日本語などのマルチバイト文字を使用しているファイルで問題が起きた場合、そのファイルが「BOMなしUTF-8」で正しく保存されているか確認してください。
*   Windows環境では意図せずShift-JISなどで保存されることがあるため、怪しい場合はファイルを一度Unicodeエスケープに置き換えるか、UTF-8で保存し直す必要があります。
